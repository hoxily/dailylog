2021年07月09日，星期五，杭州

罗元雷最近在想，怎样才能做出一个好产品。有什么样的标准或者一般性规则。
原本我以为3D数字化提升了产业的生产效率，就可以减少人员，解放出来的那些人就会去其他行业。但是我没有考虑到内卷问题。也就是生产效率提高，原本在这个行业里的人会尝试生产原本难以生产的东西，提升中端、高端产品的份额。结果就是整个市场升级。3D一旦普级，大家又都在同一起跑线上。
可是这正是中国政府一直以来在政策上讲的产业升级啊。
优势都是相对的。少部分先进的企业，相对大部分普通的企业。少量国家，相对大量发展中国家。
从长期、整体上看，我们的产品是促进产业升级换代。
从短期、个体上看，先接受数字化流程改造的企业和个人，会拥有更大的优势。

----

罗元雷解释他从上一家公司离职的原因：
1. 想要到杭州定居生活；
2. 相比服装行业，家装行业的成本过高，难以个人独立试错。

于是我又举出了拿着一个好想法，去资本市场拉投资的想法。并告诉他，时代变了。现在不再是单打独斗，倾家荡产搞投资的时代。但是罗元雷对于产品本身有非常强的掌控要求，不允许资本占据主动。对于他来说，这是他想要做的产品，而不是被资本推着走的产品。

visual studio 2019给如下的代码报告了警告，感觉像是一个bug：
```c
#include <stdio.h>

struct psi_t {
  const char* name;
  int value;
};

struct psi_t global_items[3] = {
  { "hello", 0 },
  { "world", 1 },
  { (const char*)-1, 0 } // 用(const char*)-1作为结束的标识。
};

int main() {
  for (struct psi_t* ptr = &global_items[0]; ptr->name != (void*)-1; ptr++) {
    printf("%s: %d\n", ptr->name, ptr->value);
  }
  return 0;
}
```
C6295: for循环定义错误：“const char *”值始终介于“-2147483648”到“2147483647”范围之间。循环将无限执行。

但是其实是OK的。循环能正常退出。
此代码使用gcc -Wall -Werror编译，一切正常。
